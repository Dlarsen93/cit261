Ponder: Team 4 Activity

Overview

This activity will finish the process of converting our site to a dynamic site.

Instructions

Complete the following assignment as a team. Designate one team member as the "main driver" and collaborate on their copy of the code. Everyone on the team should be actively engaged in writing the code and contributing to the solution. Once the solution is working, make sure that everyone on the team gets a copy of the code. Each week let someone else be the "main driver" of the coding.

Core Requirements

  1. The driver should visit the team's copy of the Trello board for the project. Add each of the attending team members to the "Team4: Dynamic header and footer" task...then move it to 'Doing'. Read the details of the card together.

    The driver should make sure to pull any changes from Github before proceeding. Next they should create a new branch called initials--team4. So if the driver's name were John Doe the branch should be called jd--team4

  2. Plan

    We currently have several HTML pages: index, cart, a product page for each product, and a mostly empty checkout page. What modules should we create? Which pages (views) should be entirely generated through Javascript and which would benefit from loading an HTML template in?

    • I would recommend having a one to one relationship with views/pages and modules. Then any functionality that needs to be shared between modules can be placed in a utility modules.
    • The following views are needed: productList, productDetail, cart, checkout
    • productList and productDetail are going to be mostly dynamic, so let's just generate those entirely through Javascript.
    • cart and checkout will have more static content on them...so let's load in an html partial as a starting point, and then modify as needed with Javascript.

  3. Navigation

    With a SPA links do not work the way they normally should. Normally when you click on a link it makes a request to the host server for a new file. We instead need to hijack that request and run some Javascript locally.

  4. Cart and Checkout

    Write a function to output the html to display a list of products to the home (index.html) page. The function should take the list of products as an argument, and insert the HTML markup into the DOM. It should leverage the function writted in the last step as well.

Stretch Goals

  1. Product pages

    Write a function in main.js that will filter our list of products to just the 4 we currently need. Use that function to show only those 4 tents.
